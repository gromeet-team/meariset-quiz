# 메아리셋 퀴즈 바이럴 리빌드 플랜

## 현재 문제

- 지금 구조는 `점수 합산 -> 유형 1개 출력 -> 상품 제안` 흐름이라 결과가 나를 정확히 찌르는 느낌이 약하다.
- 결과 페이지가 "설명" 위주라 캡처하고 싶은 한 줄이나 친구에게 던질 이유가 약하다.
- 실행 팁도 추상적이라 "이거 진짜 도움된다"는 체감이 낮다.

## 벤치마크 레퍼런스

1. BuzzFeed Personality Quiz
   - 참고: https://www.buzzfeed.com/tag/personality-quiz
   - 강점: 첫 문장부터 가볍고 즉시 반응 가능한 소재를 던지고, 결과는 짧고 강한 자기서사로 포장한다.

2. uQuiz
   - 참고: https://uquiz.com
   - 강점: 결과 이름이 밈처럼 짧고 세며, "나를 이상하게 정확히 긁는 문장" 하나로 저장 욕구를 만든다.

3. 16Personalities
   - 참고: https://www.16personalities.com
   - 강점: 유형 이름만 주지 않고 강점, 약점, 관계, 커리어처럼 "쓸모 있는 후속 해석"을 길게 붙여 결과의 유용성을 만든다.

4. New York Times Dialect Quiz
   - 참고: https://www.nytimes.com/interactive/2013/12/20/sunday-review/dialect-quiz-map.html
   - 보강 참고: https://peabodyawards.com/nytimesdialectquiz/
   - 강점: 결과가 "나에 대한 이야기"처럼 느껴지고, 친구와 지역/차이를 비교하고 싶게 만든다. Peabody 소개 기준으로 수천만 방문을 만들 정도로 자기 동일시가 강했다.

5. Spotify Wrapped
   - 참고: https://newsroom.spotify.com/2024-12-04/the-art-and-science-behind-spotify-wrapped/
   - 강점: 개인화된 요약, 단계적 공개, 스토리형 슬라이드, 바로 공유 가능한 카드로 "내 결과를 보여주는 행위 자체"를 콘텐츠로 만든다.

## 공통 패턴

- 결과 제목이 짧고 밈처럼 세다.
- 한 줄만 읽어도 "이거 완전 나다"가 와야 한다.
- 결과 이후에 세부 해석이 이어져서 쓸모가 생긴다.
- 친구와 비교할 소재가 분명하다.
- 공유용 카드가 결과 페이지의 부속물이 아니라 핵심 산출물이다.
- "다음에 뭘 하면 되는지"가 구체적이어야 저장 가치가 생긴다.

## 이번 리빌드에서 채택할 패턴

1. 합산형 점수 대신 `행동 프로필 분석형` 결과
   - 이유: 총점 하나보다 `시작 지연`, `주의 분산`, `유지력`, `외부 자극 반응`처럼 축이 보여야 결과가 더 정확하고 유용해진다.

2. 결과 카드에 `캡처용 판결 한 줄` 배치
   - 이유: BuzzFeed/uQuiz식 즉시 반응 포인트가 있어야 공유가 생긴다.

3. 결과 페이지를 `웃김 + 진단 + 처방` 3단 구조로 재편
   - 이유: 재미만 있으면 휘발되고, 정보만 있으면 안 퍼진다. 둘을 같이 줘야 한다.

4. Spotify Wrapped식 개인화 요약 블록 도입
   - 이유: 내 데이터를 읽어준 느낌이 나야 저장하고 보여주고 싶어진다.

5. 16Personalities식 실전 섹션 추가
   - 이유: 유형 설명만 말고 `망하는 트리거`, `맞는 환경`, `24시간 액션`, `회복법`까지 붙여 실제 도움을 만든다.

6. 친구에게 던지는 비교 문구를 결과 내부 핵심 CTA로 승격
   - 이유: 공유를 부탁하는 대신, 보내고 싶게 만들어야 한다.

## 제품 방향

- 랜딩은 "의지박약 테스트"보다 `왜 시작 직전에 자꾸 무너지는지 까발리는 테스트`로 재프레이밍한다.
- 결과는 단순 유형명이 아니라 `캐릭터 이름 + 판결 문장 + 프로필 차트 + 실전 처방`으로 제공한다.
- 캡처 카드에는 아래 3개를 반드시 넣는다.
  - 강한 별명
  - 친구가 웃을 한 줄
  - 실제 도움이 되는 핵심 처방 1개
- 비교 CTA는 "친구랑 해보세요"가 아니라 `이거 보내면 반응 오는 문장` 중심으로 설계한다.

## 성공 기준

- 결과 상단 1스크린만 캡처해도 공유 이유가 생긴다.
- 사용자가 결과에서 "왜 이 패턴이 생기는지"와 "오늘 뭘 바꾸면 되는지"를 바로 이해한다.
- 친구에게 링크를 보내는 문장이 결과 안에 이미 완성돼 있다.
